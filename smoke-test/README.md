# iOS Safari smoke test — wayfinder ticket 0011

> **THROWAWAY platform probe.** Not a game. Its only job is to let Fiachra *see
> with his eyes*, on a real iPhone, whether a single-threaded Godot 4.3 web
> export behaves the way ticket **0002** predicted — before the web-first
> platform bet is treated as fully de-risked. When 0011 resolves, this folder is
> deleted (like `prototype/` after 0004).

It is deliberately separate from `prototype/` (the 0004 dig-feel throwaway):
different question, different lifetime. It reuses the same proven export + Pages
machinery, not the same scene.

## What it proves, and how you can see it

Everything is surfaced **on-screen** — no Safari devtools needed:

| 0002 claim | What the harness does | What you look for |
| --- | --- | --- |
| Touch works (§2) | A cyan gem sprite follows your finger | Drag → the gem tracks your thumb |
| Audio unlocks on gesture, Sample playback (§3) | A one-shot blip (`AudioStreamPlayer`, `PLAYBACK_TYPE_SAMPLE`) fires on every tap | First tap makes a clean blip; no garble |
| WebGL2 / Compatibility renderer up (§2) | Reads `OS.get_current_rendering_method/driver` | Top line shows **"WebGL2 OK / gl_compatibility / opengl3"** |
| Memory is the ceiling, not CPU (§4) | Live **FPS**, **WASM linear heap** (MB + peak), **Godot static** memory (MB + peak), and a **resize counter** | ~60 FPS; memory flat, not climbing; no tab crash |
| Canvas-resize / context-loss crash (§4) | Counts every resize; a JS `webglcontextlost` listener flips a big red banner | Rotate repeatedly → counter climbs, memory stays flat, **no red banner** |

The **WASM heap** figure is read via `JavaScriptBridge` from
`window.__godotWasmMemory.buffer.byteLength` — this works on Safari, unlike
Chrome-only `performance.memory` (which shows `n/a (Safari)`).

## The iOS-safe export settings (0002 §8) — where each one lives

- **Renderer: Compatibility / WebGL2** — `project.godot` `rendering/renderer/rendering_method="gl_compatibility"` (forced on web anyway).
- **Thread Support: OFF** — `export_presets.cfg` `variant/thread_support=false`. This is *the* toggle: no `SharedArrayBuffer`, no COOP/COEP headers, dodges Safari's threaded-WASM bugs.
- **PWA: ON** — `export_presets.cfg` `progressive_web_app/enabled=true`, `display=1` (standalone / chrome-less WebClip), plus the **144 / 180 / 512** icons (`icons/`, generated by `gen_icons.py`). Godot writes the manifest + service worker.
- **Audio: Sample** — `project.godot` `audio/general/default_playback_type.web=2`, also forced on the player node.
- **Textures: Basis Universal for mobile** — `export_presets.cfg` `vram_texture_compression/for_mobile=true` + `project.godot` `textures/vram_compression/import_etc2_astc=true`.
- **Modest WASM max memory** — the `<script>` in `export_presets.cfg` `html/head_include` (readable source + full reasoning in `web/head_include.html`) clamps `WebAssembly.Memory` maximum to **512 MB**. See that file for *why* 4.3 has no preset knob and why 512 (not 256) MB.
- **Serving: HTTPS, no COOP/COEP** — GitHub Pages (see `.github/workflows/deploy-prototype.yml`).

## How to open it

- **On iPhone (the point):** open the Pages URL at **`…/smoke/`** in Safari, then
  again via **Share → Add to Home Screen** and launch the installed icon. Run the
  numbered checklist on the 0011 ticket.
- **Desktop sanity (dev):** same URL in Chromium — mouse acts as touch
  (`emulate_touch_from_mouse`). Confirms it loads/renders/inputs/plays; it does
  **not** stand in for the iPhone memory/safe-area behaviour (0002 §8 — the
  simulator doesn't either).
- **In the editor:** open `project.godot` in Godot 4.3+ and press play.

## Files

- `scripts/Smoke.gd` — the whole probe: audio, renderer probe, JS-bridge memory
  reads, touch sprite, resize counter, drawn in `_draw()`.
- `scenes/Main.tscn` — one node, the script attached.
- `export_presets.cfg` — the iOS-safe **Web** preset (see table above).
- `web/head_include.html` — readable source of the WASM-memory clamp shim.
- `icons/` + `gen_icons.py` — the PWA icons and their pure-stdlib generator.
- `../.github/workflows/deploy-prototype.yml` — CI: exports prototype **and**
  this smoke test, publishes prototype at `/` and smoke test at `/smoke/`.
